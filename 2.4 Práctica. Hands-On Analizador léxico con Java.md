# Práctica: Analizador Léxico en Java

## Descripción

Se implementó un analizador léxico en Java capaz de reconocer tokens como palabras clave, identificadores, literales, operadores y separadores a partir de código fuente Java. Además, detecta errores léxicos y continúa el análisis mediante recuperación de errores.

# Código del Analizador Léxico

```java
import java.util.regex.*;
import java.util.*;
import java.nio.file.*;
import java.io.IOException;

public class KeywordsIdentifiers {

    // Tipos de token
    enum TokenType {
        FLOATLIT, HEXADECIMALLIT, BINARYLIT, OCTALLIT, INTEGERLIT, STRINGLIT, CHARLIT, KEYWORD, IDENTIFIER, OPERATOR,
        SEPARATOR, WHITESPACE, UNKNOWN
    }

    // Record para representar un Token (Java 16+)
    record Token(TokenType type, String lexeme, int line) {
        public String toString() {
            return String.format("[L%-3d] %-14s → \"%s\"", line, type, lexeme);
        }
    }

        /**
     * Excepción para errores léxicos — contiene línea y columna exactas
     */
    public static class LexicalError extends RuntimeException {
        private final int line, column;
        private final char illegal;

        public LexicalError(char c, int line, int col) {
            super(String.format(
                "Error léxico [línea %d, col %d]: carácter ilegal '%c' (U+%04X)",
                line, col, c, (int) c
            ));
            this.line = line; this.column = col; this.illegal = c;
        }
    }
    static List<LexicalError> errors = new ArrayList<>();
    // Patrón de keywords (todas las palabras reservadas de Java)
    static final String KW_PATTERN = "\\b(abstract|assert|boolean|break|byte|case|catch|char|" +
            "class|const|continue|default|do|double|else|enum|extends|" +
            "final|finally|float|for|if|implements|import|instanceof|" +
            "int|interface|long|new|package|private|protected|public|" +
            "return|short|static|super|switch|synchronized|this|throw|" +
            "throws|try|void|volatile|while|true|false|null)\\b";

    // Patrón MASTER con grupos nombrados
    static final Pattern MASTER = Pattern.compile(
        // 1. Comentarios primero (no producen tokens de salida)
        "(?<JAVADOC>/\\*\\*[\\s\\S]*?\\*/)|"   +
        "(?<BLOCKCOMMENT>/\\*[\\s\\S]*?\\*/)|"     +
        "(?<LINECOMMENT>//[^\\n]*)|"              +
        // 2. Literales de cadena y char (contienen cualquier carácter)
        "(?<STRINGLIT>\"([^\"\\\\]|\\\\.)*\")|"      +
        "(?<CHARLIT>'([^'\\\\]|\\\\.)')|"         +
        // 3. Literales numéricos (FLOAT > HEX/BIN/OCT > INT)
        "(?<FLOATLIT>[0-9][0-9_]*\\.[0-9][0-9_]*([eE][+-]?[0-9]+)?[fFdD]?)|" +
        "(?<HEXADECIMALLIT>0[xX][0-9a-fA-F][0-9a-fA-F_]*[lL]?)|" +
        "(?<BINARYLIT>0[bB][01][01_]*[lL]?)|"       +
        "(?<OCTALLIT>0[0-7]+[lL]?)|"               +
        "(?<INTEGERLIT>0|[1-9][0-9_]*[lL]?)|"        +
        // 4. Keywords antes que dentificadores
        "(?<KEYWORD>" + KW_PATTERN + ")|"           +
        "(?<IDENTIFIER>[a-zA-Z_$][a-zA-Z0-9_$]*)|" +
        // 5. Operadores compuestos antes que simples
        "(?<OPERATOR>==|!=|<=|>=|&&|\\|\\||<<|>>>|>>|\\+\\+|--|\\+=|-=|\\*=|/=|[+\\-*/%<>=!&|^~?:])|" +
        // 6. Separadores
        "(?<SEPARATOR>[(){}\\[\\];,.])|"             +
        // 7. Whitespace (descartar)
        "(?<WHITESPACE>[ \\t\\r\\n]+)",
        Pattern.MULTILINE 
        );

    // Método de Tokenización
    public static List<Token> tokenize(String source) {
        List<Token> tokens = new ArrayList<>();
        Matcher m = MASTER.matcher(source);
        int line = 1, pos = 0;

        while (m.find()) {
            // Detectar caracteres no reconocidos
            // En el tokenizador — estrategia PANIC RECOVERY:
            // Si queda texto sin consumir, avanzar 1 char, registrar el error y continuar.

            if (m.start() > pos) {
                String illegal = source.substring(pos, m.start());
                int col = pos - source.lastIndexOf('\n', pos);
                System.err.printf("Error léxico [L%d, C%d]: '%s'%n", line, col, illegal);
                errors.add(new LexicalError(illegal.charAt(0), line, col));
            }
            if (m.group("WHITESPACE") != null) {
                // Contar saltos de línea en el whitespace
                line += m.group("WHITESPACE").chars().filter(c -> c == '\n').count();
            } else if (m.group("FLOATLIT") != null)
                tokens.add(new Token(TokenType.FLOATLIT, m.group(), line));
            else if (m.group("HEXADECIMALLIT") != null)
                tokens.add(new Token(TokenType.HEXADECIMALLIT, m.group(), line));
            else if (m.group("BINARYLIT") != null)
                tokens.add(new Token(TokenType.BINARYLIT, m.group(), line));
            else if (m.group("OCTALLIT") != null)
                tokens.add(new Token(TokenType.OCTALLIT, m.group(), line));
            else if (m.group("INTEGERLIT") != null)
                tokens.add(new Token(TokenType.INTEGERLIT, m.group(), line));
            else if (m.group("STRINGLIT") != null)
                tokens.add(new Token(TokenType.STRINGLIT, m.group(), line));
            else if (m.group("CHARLIT") != null)
                tokens.add(new Token(TokenType.CHARLIT, m.group(), line));
            else if (m.group("KEYWORD") != null) {
                tokens.add(new Token(TokenType.KEYWORD, m.group(), line));
            } else if (m.group("IDENTIFIER") != null) {
                tokens.add(new Token(TokenType.IDENTIFIER, m.group(), line));
            } else if (m.group("OPERATOR") != null) {
                tokens.add(new Token(TokenType.OPERATOR, m.group(), line));
            } else if (m.group("SEPARATOR") != null) {
                tokens.add(new Token(TokenType.SEPARATOR, m.group(), line));
            }else if (m.group("JAVADOC") != null || m.group("BLOCKCOMMENT") != null) {
              line += m.group().chars().filter(c -> c == '\n').count();
            }
            pos = m.end();
        }
        return tokens;
    }



public static void main(String[] args) {
    try {
        String path = "C:\\Users\\Baruchs Delgado\\Downloads\\reportePractica\\PruebaCompleta.java";

        // Leer todo el archivo como String
        String code = Files.readString(Path.of(path));

        // Tokenizar
        List<Token> tokens = tokenize(code);

        tokens.forEach(System.out::println);

        if (!errors.isEmpty()) {
            System.err.println("\nErrores léxicos:");
            errors.forEach(e -> System.err.println(e.getMessage()));
        }

    } catch (IOException e) {
        System.err.println("Error al leer el archivo: " + e.getMessage());
    }
}

}
```

# Preguntas de Discusión

### 1. ¿Qué pasa si eliminas los `\b` del patrón KEYWORD?

Se reconocerían palabras clave dentro de identificadores, por ejemplo int dentro de integer, lo cual es incorrecto.

---

### 2. ¿Por qué KEYWORD debe evaluarse antes que IDENTIFIER?

Porque las palabras clave cumplen la estructura de identificador. Si no se priorizan, nunca serían reconocidas como keywords.

---

### 3. ¿Se puede construir una sola ER para ambos tokens?

Sí, pero no es recomendable. Es mejor reconocer identificadores y luego validar si son keywords.

---

### 4. ¿Cómo manejarías el operador ternario `?:`?

Se reconoce como dos operadores individuales (`?` y `:`). El parser se encarga de interpretarlo.

---

### 5. ¿Por qué `>>>` debe ir antes que `>>`?

Para evitar que se divida incorrectamente (`>>>` - `>>` - `>`). Se aplica la regla de coincidencia más larga.

---

### 6. Diferencia entre separador y operador

* Separadores: estructuran el código (`{ } ; ,`)
* Operadores: realizan operaciones (`+ - == &&`)

---

### 7. ¿Por qué `08` no es octal válido?

Porque los números octales solo permiten dígitos del 0 al 7.

---

### 8. ¿Cómo soportar Text Blocks (`"""`)?

Agregando un patrón adicional antes de `STRINGLIT` que capture bloques multilínea.

---

### 9. ¿Debe el lexer validar rangos numéricos?

No, eso corresponde al análisis semántico, no al léxico.

---

### 10. ¿Cómo manejar comentario no cerrado?

Detectar el error y reportarlo, indicando la línea donde inicia.

---

### 11. ¿JavaDoc debe ser token diferente?

Depende para documentación sí, para compilación básica no es necesario.

---

### 12. Diferencia entre errores en lexer y parser

* Lexer: errores de caracteres inválidos
* Parser: errores de estructura sintáctica

# ⚙️ Decisiones de Diseño

## 1. Uso de Expresiones Regulares

Se eligió usar un **patrón maestro (MASTER)** con grupos nombrados porque:

* Permite reconocer múltiples tokens en una sola pasada
* Mejora la eficiencia
* Facilita el mantenimiento del código

---

## 2. Prioridad de Tokens

Se ordenaron los patrones considerando:

* Keywords antes que identificadores
* Operadores largos antes que cortos
* Comentarios antes que cualquier otro token

Esto evita ambigüedades en el reconocimiento.

---

## 3. Lectura de Archivos

Se utilizó:

```java
Files.readString(Path.of(path));
```

Porque:

* Es simple
* Lee todo el archivo de una sola vez
* Es adecuado para pruebas y archivos pequeños/medianos

---

## 4. Manejo de Errores (Panic Recovery)

Se implementó una estrategia de:

* Detectar caracteres no reconocidos
* Registrar el error
* Continuar el análisis

Esto permite que el lexer:

* No se detenga ante errores
* Analice el resto del archivo


---

## 5. Salida en Consola

Se decidió imprimir:

* Tokens reconocidos
* Errores léxicos al final

Esto permite:

* Verificación rápida
* Depuración sencilla
